# プロンプトの書き方 ― 文字描画の鉄則とモデル別の違い

（情報は 2026 年時点。モデルは更新が速いので、必要なら最新仕様を確認する。）

## 文字描画の鉄則（モデル共通・必須）

1. **入れる文字を一字一句列挙する。** 抽象指示（「○○を説明する図」）は英語混入・文字化けの原因。表示したい日本語を正確に書く。
2. **「画像に描く文字は上記で挙げたものだけ。指定外の文字・ラベル・英単語は描かない」** をプロンプト末尾に必ず入れる。
3. **「日本語で」などの冗長語を足さない。** テキスト内容を正確に与えることが効く。
4. **「漢字・ひらがな・カタカナを正確に、文字化けや誤字なく描く」** と添える。
5. 長文は避け、**短いラベル**にする。文字が多いほど崩れやすい。

## 内容の忠実さ

- 文言は素材（概念・文書）に基づく事実のみ。固有名詞・数値は素材で確認できるものだけ。
- 創作の比喩・情緒・誇張語を入れない。

## Nano Banana Pro（Gemini 3 Pro Image）

- 日本語の描画が大幅に改善され、ひらがな・カタカナ・漢字をデザインに馴染ませて正確に描ける。図解・ホワイトボード風が得意。
- **ゾーンを箇条書きで明示した設計図的プロンプト**が効きやすい（【上部】【中央】…のように領域ごとに文言を指定）。
- **比率はプロンプト内に**「横長16:9」と書く。
- **参照画像（最大14枚）**で絵柄・キャラ・配色を保てる。シリーズ化するなら、よい1枚を作って参照画像にし、プロンプトの一部だけ差し替える。
- 提供：Gemini アプリ / Google AI Studio / API。

## GPT Image 2（OpenAI）

- テキスト描画が非常に強く、図解にも向く。高解像度。
- **一段落の制作ブリーフ形式**が馴染む（「〜なインフォグラフィックを作成してください。上部に…、中央に…」と散文で書く）。
- **比率・解像度はパラメータで指定**（例：`size: "1536x1024"` 横長、`quality: "high"`）。プロンプトには「横長」程度を添える。
- ChatGPT 上なら、生成後に**会話で部分修正**できる（「○○の文字を直して」「ロードマップを省いて」）。
- 端正に寄りがちなので、**「手描き・マーカー・ラフ・線を不揃いに」**を強めに指定すると親しみやすくなる。

## 仕上げ・トラブル対応

- 文字が崩れた箇所は、**部分再生成（インペイント）**で直すか、ChatGPT 上なら会話で修正。
- 情報過多で潰れるなら、ゾーンを削る／コンパクト版を使う。
- 一連の図解（章扉など）は、スタイル指定を固定し、中央ゾーンのテーマだけ差し替える。
