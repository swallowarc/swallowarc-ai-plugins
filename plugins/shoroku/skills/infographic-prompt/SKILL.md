# Infographic Prompt（手書き風説明画像のプロンプト生成）

## Overview

概念・文書・本の要点を、**手描きホワイトボード解説風の1枚絵インフォグラフィック**にするための、画像生成プロンプトを作るスキル。出力は最新の文字描画に強い画像モデル（**Nano Banana Pro / Gemini 3 Pro Image**、**GPT Image 2**）向けのプロンプト。

**中核原則：画像に入れる文字は、与えられた素材に基づく正確なものだけ。** 創作の事実・比喩を足さない。読みやすさ（legibility）を最優先し、詰め込みすぎない。

## When to use

- 「この内容を手描き風の1枚絵にしたい」「Nano Banana / GPT で図解する画像のプロンプトを作って」
- 記事・本・概念・自作ドキュメント（要約HTML等）を、ゆるかわインフォグラフィックにまとめたいとき

使わない場面：実際に画像を生成する作業そのもの（このスキルは**プロンプトを作る**まで）。HTML/CSS の要約サイトを作るなら別スキル（book-summary-html）。

## 文字描画の鉄則（最重要・モデル共通）

1. **画像に入れる文字を一字一句プロンプトに列挙する。** 「○○を説明する図」のような抽象指示は、英語混入・文字化けの原因。
2. **「指定した文字以外は描かない」と明記する。** これを必ずプロンプト末尾に入れる。
3. **「日本語で」などの冗長な語を足さない**（逆効果になりうる）。
4. **漢字・ひらがな・カタカナを正確に**描くよう指示する。
5. 文字数が多すぎると潰れる。**1枚に詰め込みすぎない**。多いならゾーンを削るか「コンパクト版」を併記する。

## 内容の鉄則

- 画像の文言は**素材（ユーザ提供の概念・文書）に基づく事実**のみ。数値・固有名詞は素材で確認できるものだけ。
- 創作の比喩・情緒的表現・誇張語（「劇的に」等）を入れない。
- 要約対象がある場合、**核となる1メッセージ**を決め、それを主役（中央）に据える。枝葉は周辺ゾーンへ。

## Process

1. **素材と狙いを確認する。** 何を1枚にするか（概念／文書／本の要点）、対象読者、入れたい要素。曖昧なら核メッセージとラベルをユーザに確認する（文字は正確である必要があるため）。
2. **核メッセージを1つに決める。** 中央の主役にする。
3. **ゾーンに割り付ける。** `references/style-guide.md` のゾーン例（タイトル／中央の主役／対比 before-after／指標カード／手順ステップ／ロードマップ／マスコット／下部キャプション）から、内容に合うものを選ぶ。無理に全部使わない。
4. **入れる文字を全列挙する。** 各ゾーンに表示する文言を確定する（素材に忠実に）。
5. **プロンプトを書く。** `templates/prompt.md` を雛形に、**Nano Banana Pro 版**と **GPT Image 2 版**を作る。スタイルは `references/style-guide.md`、モデル別の差は `references/prompt-rules.md` に従う。
6. **出力する。** 既定では1つの md ファイルに両モデル版＋コンパクト版＋配色メモ＋仕上げ手順をまとめる。ユーザが望めばモデルごとに別ファイルに分ける。

## スタイル（手描き説明画像）の要点

`references/style-guide.md` に詳細。骨子：

- 手描きマーカースケッチ調、クリーム色の紙背景、ラフな角丸枠、蛍光ペンの下線、パステル配色
- 手書き風の日本語フォント、かわいい丸いロボット等のマスコット
- 横長（16:9 目安）、情報は**左から右へ流れる**構成、余白をとって読みやすく

## モデル別の要点

`references/prompt-rules.md` に詳細。

- **Nano Banana Pro**：ゾーンを箇条書きで明示した設計図的プロンプトが効く。比率はプロンプト内に「横長16:9」。参照画像（最大14枚）で絵柄を統一できる。
- **GPT Image 2**：一段落の制作ブリーフ形式が馴染む。**比率・解像度はパラメータ**（例 `size: "1536x1024"`）。ChatGPT 上なら会話で部分修正が速い。端正になりがちなので「手描き・ラフ」を強めに。

## Common mistakes

- **入れる文字を列挙しない** → 英語混入・文字化け。必ず全列挙。
- **「指定外の文字を描かない」を省く** → 余計な文字が湧く。
- **1枚に詰め込みすぎる** → 文字が潰れる。ゾーンを削る／コンパクト版を作る。
- **素材にない事実・比喩を足す** → 内容の鉄則違反。素材に忠実に。
- **モデル差を無視** → GPT は比率パラメータ、Nano Banana は参照画像での一貫性、を使い分ける。

## Files

- `references/style-guide.md` — 手描き説明画像のスタイル定義とゾーン例・配色
- `references/prompt-rules.md` — 文字描画の鉄則とモデル別（Nano Banana Pro / GPT Image 2）の書き方
- `templates/prompt.md` — 出力する md の雛形（両モデル版＋コンパクト版）
