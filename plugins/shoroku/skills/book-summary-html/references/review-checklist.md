# レビューチェックリスト（フェーズ3）

専任レビュアーのサブエージェントが、生成済み HTML 群を検査するための項目。
**役割**：生成器とは別コンテキストで、忖度せず違反を指摘する。各指摘は `ファイル名:該当箇所（見出しや文の冒頭）` を添えて報告する。問題がなければ「問題なし」と明示する。

## A. 正確さ（本への忠実さ）

- [ ] 本にない事実・数値・固有名詞・事例が、`補足（本書外）` マークなしで書かれていないか
- [ ] `補足（本書外）` で囲まれた情報が、明らかな誤りを含んでいないか
- [ ] 著者が述べていない因果関係・断定がないか
- [ ] フェーズ1の「著者の主張リスト」の各項目が、対応する章に反映されているか（**漏れの検出**）
- [ ] AI独自の分析・提案が、index の「おすすめの読み進め方」（`.ai-analysis`＋AIバッジ）**以外**の場所に混入していないか
- [ ] 「おすすめの読み進め方」ブロックに AIバッジが付き、本書外と明示されているか。提案が本の構成・主張と矛盾していないか
- [ ] 「ターゲット読者」ブロックに「（本書の内容から推定）」が添えられているか

## B. 文体・用語の統一

- [ ] 文末がだ・である調で全ページ統一されているか（です・ます調の混在を検出）
- [ ] 同一概念の用語が統一されているか（表記ゆれ・言い換えを検出）
- [ ] 翻訳調・直訳調の不自然な日本語がないか（不自然な受動態、逐語訳的言い回し）
- [ ] 不要な主語・指示語の多用（「あなたは」「我々は」「それは〜である」等）がないか

## C. 中身のない文章

- [ ] つなぎの空文（情報を持たない前置き・「重要である」だけの文）がないか
- [ ] 要点（見出し）と解説が同義反復になっていないか
- [ ] 同じ内容の重複がないか

## D. 比喩・情緒・主張

- [ ] 本にない比喩・たとえ話がないか
- [ ] 情緒的・感情的・ポエム的表現がないか
- [ ] 書き手の主張・評価・提言が混ざっていないか

## E. グラフィカル表現

- [ ] 絵文字が本文（解説文）の中に混ざっていないか（UI アイコン枠のみ許容）
- [ ] 図解部品内の文言が A〜D のルールに違反していないか
- [ ] 本にない誇張語（劇的に・圧倒的に等）がないか
- [ ] カスタム図（SVG 等）が、本書にある情報のみを表しているか（本にない関係・分類・数値・主張を図にしていないか）
- [ ] カスタム図の本書由来でない視覚要素（識別用の配色・配置）が `figcaption` 等で明示されているか
- [ ] カスタム図の乱用がないか（既存部品で足りる内容を図にしていないか／同じ構造を複数の図で繰り返していないか／装飾・箇条書きの焼き直し・要点の水増しの図がないか）

## F. 構造・整合

- [ ] index 冒頭に「ご利用前の注意」固定文（`.about-notice`・黄色基調）があり、文言が改変されていないか（AIによるWebリサーチ生成・書籍と一致しない可能性・非網羅・読書サポート用途が明記されているか）
- [ ] index に書誌情報が固定書式で記載されているか
- [ ] ページ間リンク（前後の部/章・目次）が正しく解決するか
- [ ] 各ページの文体・部品の使い方が揃っているか（ページ間のばらつき）

## G. 詳細モード固有（mode=詳細のときのみ適用）

標準モードでは適用しない。レビュー依頼で詳細モードと伝えられたときだけ検査する。

- [ ] 論拠展開段落が、要点の言い換えでなく**新しい情報（根拠／前提／筋道）**を持っているか（水増しの検出）
- [ ] 展開された論拠が**出典・本書で裏取りできる範囲**にとどまっているか（推測の混入の検出）
- [ ] `補足（本書外）` が**専門用語の短い定義**に限定され、背景・論評の拡張に使われていないか
- [ ] 比喩・情緒・著者にない主張が、深掘りに紛れて混入していないか
- [ ] 深さを口実に**トピックや図が水増し**されていないか（詳細モードでも見出し・トピックの骨格は標準と同じはず）

## 報告フォーマット

```
## レビュー結果
### 重大（要修正）
- partX.html「見出し名」: 〔違反内容〕

### 軽微（任意）
- ...

### 著者の主張カバレッジ
- 第N章: 主張「…」が未反映 / 反映済み
```
