# 文章ルール（book-summary-html）

要約本文を書くときに守る規則。フェーズ2で適用し、フェーズ3で検査する。

## 1. 正確さ（最優先）

- **本にない事実は書かない。** 数値・固有名詞・事例・因果関係は、本の記述で確認できるものだけを書く。
- 読者の理解のために補足が有益で、かつ**誤りでないと確認できる**場合に限り、本書外の情報を書いてよい。その際は必ず `補足（本書外）` 部品（`style-components.md` 参照）で囲み、本書由来でないと視覚的に分かるようにする。
- 不確かなら書かない。推測を断定で書かない。
- 著者の主張と、一般論・自分の解釈を混ぜない。

### 例外：index の「おすすめの読み進め方」

- index の「おすすめの読み進め方」ブロックに限り、**AI が本の内容を分析した独自の提案**を書いてよい（本にない情報の唯一の例外）。
- 必ず `.ai-analysis` ＋ AIバッジ（「AIによる分析 ― 本書の記述ではなく、内容に基づく独自提案」）で囲み、本書の記述と明確に区別する。
- 提案内容は本の構成・主張と整合する範囲にとどめ、事実誤りを書かない。
- この例外は当該ブロック以外には一切適用しない。
- なお「ターゲット読者」ブロックは本の内容からの**推定**であり、本書外の創作ではない。AIバッジは付けず、末尾に「（本書の内容から推定）」と添える。

## 2. 文体・用語の統一（AI臭の防止）

- **文末はだ・である調に統一する。** 「〜です／ます」「〜だ／である」を混在させない。体言止めは見出し・箇条書きの要点では可、解説文では多用しない。
- **用語を統一する。** 同じ概念を別語で言い換えない（例：「リードタイム」を途中から「所要時間」にしない）。原書の訳語に合わせる。
- **翻訳調・直訳調を避ける。** 不自然な受動態、英文を逐語訳したような言い回し、二重否定の乱用をしない。
- **不要な主語を明示しない。** 「あなたは〜」「我々は〜」「それは〜である」のような、日本語として不要な主語・指示語の多用を避ける（翻訳調の典型）。
- 一文を長くしすぎない。一文一義を基本とする。

## 3. 中身のない文章の禁止

- **つなぎの空文を書かない。** 「ここでは〜について見ていく」「〜は非常に重要である」だけで情報を持たない文は削る。
- 同じことを言い換えただけの重複を避ける。要点と解説で内容が重ならないようにする。
- 見出しと解説が同義反復になっていないか確認する。

## 4. 比喩・情緒・主張の禁止

- 本にない**比喩・たとえ話**を入れない（例として作った「鎖の弱い輪」「氷山の一角」などは、本書に出てこなければ使わない）。
- **情緒的・感情的表現、ポエム的表現**を入れない（「驚くべきことに」「まさに〜と言えよう」など）。
- 著者が述べていない**主張・評価・提言**を書かない。要約は著者の主張を伝えるもので、書き手の意見を述べる場ではない。

## 5. 著者の主張を漏らさない

- フェーズ1で抽出した「各章の著者の主張」リストを手元に置き、各章の要約がそれを**すべて含む**ことを確認する。
- 要点（見出し）は、著者が立てた論点・主張に対応させる。枝葉のディテールより主張の骨格を優先する。

## 6. グラフィカル表現の節度

- 絵文字は**UI のアイコンとしてのみ**使う（コールアウトや見出しのアイコン枠）。**本文（解説文）の中に絵文字を混ぜない。**
- 図解部品（対比カード・ステップ・指標カードなど）に入れる文言も、上記ルール（本にない情報を足さない・比喩を使わない）に従う。
- 装飾のための誇張（「劇的に」「圧倒的に」等、本にない強調語）を足さない。

## セルフチェック（書いた直後に）

- [ ] 文末はだ・である調で揃っているか
- [ ] 本にない情報に `補足（本書外）` が付いているか／そもそも不要な補足を足していないか
- [ ] 比喩・情緒・ポエム・自分の主張が混ざっていないか
- [ ] 中身のないつなぎ文・重複がないか
- [ ] 各章の著者の主張をすべて反映したか
- [ ] 用語の表記ゆれがないか
