# AskUserQuestion ダイアログ表示時に直前のテキストがユーザーに表示されない

- 記録日: 2026-07-12
- 発生環境: Claude Code CLI（モデル: claude-fable-5）
- 関連プラグイン: kata 0.1.1（`kata:design` スキル実行中に発生）
- 状態: 未解決（ワークアラウンドで回避中）

## 症状

同一ターン内で「設計セクションの本文をテキスト出力 → AskUserQuestion ツールで承認確認」という順で応答した場合、ツール呼び出しの前に出力したテキストがユーザーの画面に表示されず、確認ダイアログだけが表示された。

ユーザーには「何を確認すればよいのか分からない」状態になる。実際のやり取り:

1. アシスタントがセクション本文（ディレクトリ構造・ループ挙動などの設計内容）を出力し、続けて AskUserQuestion を呼んだ
2. ユーザーの回答: 「この内容とはどの内容？」
3. 本文を再掲して再度 AskUserQuestion を呼んでも同じ症状: 「先ほどから確認して欲しいと思われる情報が表示されていない」

なお、それ以前のセクション確認（選択肢の description に判断材料を全部書き込んでいたもの）は回答が成立していたため、**ダイアログの選択肢内テキストは見えているが、ターン途中の通常テキスト出力が見えていない**と推定される。

## 影響

`kata:design` の中核フローである逐次承認（セクションを提示 → AskUserQuestion で確認）が成立しない。ユーザーが内容を見ないまま「よい」を選んでしまう危険があり、承認ゲートとして機能しなくなる。

## ワークアラウンド（本セッションで採用）

- セクション本文は**ターンの最後のテキスト**として出力してターンを終える（後続のツール呼び出しを置かない）
- 承認はダイアログではなく、ユーザーの通常の返信でもらう

代替案として、AskUserQuestion を使う場合は確認対象の内容を選択肢の `description` / `preview` に埋め込む方法もあるが、設計セクション程度の分量では収まらない。

## kata:design スキルへの反映候補

- 「セクション提示 → 承認確認」の手順に、承認確認をダイアログで行う場合は**提示テキストと同一ターンに置かない**（テキスト提示でターンを終え、返信で承認を受ける）よう明記する
- あるいは AskUserQuestion 利用時は確認対象を preview に必ず埋め込める粒度まで小さくする、という制約を明記する
