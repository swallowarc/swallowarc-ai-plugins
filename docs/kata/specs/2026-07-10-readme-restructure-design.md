# README.md 再構成 設計 spec

- 日付: 2026-07-10
- 対象: `README.md`（本リポジトリのルート）
- ステータス: ユーザー承認済み設計の文書化

## 背景と目的

現在の README は利用者向け情報（インストール方法・収録スキル）と開発者向け情報（リポジトリ構成・バージョン更新ルール・フック）が交互に並んでおり、どの読者に向けた文書なのかが中途半端になっている。

これを「まず利用者向け、その後に開発者向け」の 2 部構成に再編する。想定読者は **GitHub でこのリポジトリを見つけた第三者を含む**。前提知識なしで導入できる書き方とし、個人環境依存の記述は汎用化する。

作業範囲は **並べ替え＋文言修正**。全面書き直しはしない。既存の記述は原則維持し、変更点はこの spec に列挙したものに限る。

## イントロ＋利用ガイドの詳細

### イントロ（見出しなし、冒頭）

- 現行の説明文（AI エージェント向けプラグイン管理・配布リポジトリ、Claude Code のプラグインマーケットプレイスとして機能する）と、配布プラグイン 2 つ（`swallowarc`・`kata`）の箇条書きを維持する。
- 開発者視点の一文「新しい単発スキルはここに追加していく」は**イントロから削除し、開発ガイド「リポジトリ構成」へ移動**する。

### インストール

- 現行「利用方法（Claude Code）」の `marketplace add` / `plugin install` の説明を移設する。
- ローカルで試す場合の例 `/plugin marketplace add /Users/acrobat.kame/IdeaProjects/swallowarc-ai-plugins` は、個人環境依存のため `<リポジトリをクローンしたローカルパス>` の形式に汎用化する。

### 収録スキル（swallowarc）

- 表（スキル / 呼び出し / 説明）は維持する。
- `book-summary-html` の説明は現在 mode の詳細まで含む長文になっているため、**1〜2 文に簡潔化**し、詳細は SKILL.md 側に委ねる。
  - トレードオフ: README 単体でスキルの全機能は分からなくなるが、SKILL.md との二重管理を減らせる。スキル詳細の正は SKILL.md とする。

### 収録スキル（kata）

- 表（スキル / 原典 superpowers / 説明）と基本フロー（design → plan → execute → … → finish、バグ対応は debug から）を維持する。原典列は出自の明示として残す。
- 「設計の詳細は `docs/kata/specs/` を参照」という記述は、**参照先である kata スキル群の設計文書が存在しない stale な参照のため削除**する（本 spec の保存で `docs/kata/specs/` ディレクトリ自体は作られるが、そこに置かれるのは README 再構成の spec であり、kata の設計文書ではない）。

### ディスパッチャー（using-kata）の有効化

- 現行内容（CLAUDE.md 方式／SessionStart フック方式の 2 案）をほぼそのまま移設する。

### 更新・削除

- `marketplace update` / `marketplace remove` のコマンドを独立サブセクションとして利用ガイド末尾に置く。

## 開発ガイド＋ライセンスの詳細

### リポジトリ構成

- 現行の「リポジトリ構成」（ディレクトリツリー）と「名前空間の注意」（blockquote）を **1 つのサブセクションに統合**し、開発ガイドの先頭に置く。
- イントロから移した「新しい単発スキルは `swallowarc` に追加する」という方針をここに記載する。

### 新しいスキルの追加手順

- 現行の 5 ステップを維持する。
- ステップ 4（plugin.json の version 更新）から「バージョン更新ルール」節へ内部アンカーリンクを張る（節の順序が現行と逆になり後方参照になるため）。

### バージョン更新ルール（必須）

- 現行内容（なぜ必須か、変更内容とバージョンの対応表、混在時はより大きい方を優先）を維持する。
- 順序は「スキル追加手順 → バージョン更新ルール」とする（現行は逆順）。

### 自動化（Claude Code フック）

- 現行内容（PostToolUse フック、settings.json とスクリプトの役割分担、フックは Markdown で宣言できない旨の注記）をそのまま開発ガイド末尾に移設する。

### ライセンス

- 開発ガイドの外、README 最末尾に現行内容（MIT License、kata プラグインは obra/superpowers のプロセス設計に基づく再構成である旨）のまま維持する。

## 検証方針

書き換え後に次の 3 点を確認する。

1. 見出し構成が本 spec の「新しい全体構成」と一致するか。
2. 内部アンカーリンク（スキル追加手順 → バージョン更新ルール）が GitHub 上で機能するアンカー形式か。
3. README が言及するパス（`.claude/settings.json`、`.claude/hooks/skill-changed.sh`、`plugins/` 配下、`.claude-plugin/marketplace.json`）が実在するか。

## バージョン更新の扱い

README のみの変更であり、スキル（SKILL.md）の追加・更新ではないため、**plugin.json の version 更新は不要**。

## スコープ外

- CONTRIBUTING.md の新設（開発者向け情報は README 内の後半に置く方針のため）
- SKILL.md・marketplace.json・フックスクリプトの変更
- README の全面書き直し（変更は本 spec 列挙分に限る）
